## About This Repository

This repository provides beginner-friendly and practitioner-focused manuals for:

* Cybersecurity professionals
* AI security engineers
* GRC and compliance analysts
* Managers and auditors
* Cloud security engineers
* Students and career changers
* Educators and mentors

The collection includes practical guidance covering:

* AI security and responsible AI
* AI governance, risk, and compliance
* ISO/IEC 42001 Artificial Intelligence Management Systems
* NIST AI Risk Management Framework
* OWASP guidance for generative AI and large language model applications
* MITRE ATLAS
* Retrieval-Augmented Generation security
* Model Context Protocol security
* AI agents and autonomous systems
* Prompt injection, data leakage, and model abuse
* Cloud AI security for Microsoft Azure, Amazon Web Services, Google Cloud, Oracle Cloud, and IBM Cloud
* Kali Linux and authorized security testing
* Digital forensics
* Security evidence collection and audit readiness
* Practical responsibilities for managers, analysts, and engineers

## Suggested Learning Path

New learners may find this sequence helpful:

1. Begin with AI risk management and governance fundamentals.
2. Study ISO/IEC 42001 and understand the AI management-system lifecycle.
3. Select the cloud platform most relevant to your work.
4. Learn identity, data, model, application, logging, and infrastructure security.
5. Use the Kali Linux manual only in an approved laboratory or authorized environment.
6. Practice documenting findings, evidence, risks, controls, and corrective actions.
7. Revisit the manuals as technologies, threats, standards, and legal requirements change.

## Safe and Ethical Use

All technical material in this repository is intended for lawful, authorized, and ethical use.

Use security tools and testing procedures only when you:

* Own the system or environment
* Have explicit written permission
* Are working in an approved laboratory
* Are participating in an authorized training platform
* Are operating within an approved professional engagement
* Remain within the documented scope and testing period

> **Technical ability does not create permission.**

Do not use this material to access, test, disrupt, monitor, alter, or collect information from systems, applications, AI models, accounts, networks, or data without authorization.

## Core AI Security Principles

AI systems should be designed and operated with:

* Security
* Privacy
* Accountability
* Transparency
* Human oversight
* Fairness
* Reliability
* Resilience
* Traceability
* Regulatory compliance

Security should be addressed throughout the AI lifecycle, including:

* Planning
* Data collection and preparation
* Model development or selection
* Testing and validation
* Deployment
* Monitoring
* Incident response
* Change management
* Retirement
* Secure disposal

## Versioning and File Naming

Manual filenames identify the subject, language, and release version.

Recommended format:

```text
Manual_Title_Language_v1.0.pdf
Manual_Title_Language_v1.0.docx
```

Release numbering:

* **v1.0** — Initial public release
* **v1.1** — Corrections, formatting improvements, or updated links
* **v1.2** — Added tools, examples, or implementation guidance
* **v2.0** — Major restructuring or substantial new content

The filename examples above explain the naming convention. They are not download links. Use the links in the **Download the Manuals** section to access the published documents.

## Contributions and Corrections

Corrections, accessibility improvements, and responsible contributions are welcome.

Contributions should:

1. Use accurate and authoritative information.
2. Prefer primary and official sources.
