# Kali Linux, AI Security, GRC, and Digital Forensics Learning Manual

A free, beginner-friendly collection of practical cybersecurity manuals covering Kali Linux, ethical hacking, AI security, AI governance, GRC, digital forensics, and safe hands-on learning.

**Author:** Alberto (Al) Leiva

---

## About This Repository

This repository was created to make cybersecurity learning more accessible to students, junior analysts, career changers, managers, and anyone beginning a practical journey into security.

The collection combines technical instruction with strong ethical guidance. It explains not only how tools and frameworks work, but also when they should be used, why authorization matters, how to document results, and how security activities connect to governance, risk, compliance, and professional responsibility.

The manuals are written in clear, practical language and are designed to support both self-study and structured training.

## Recommended Learning Path

A new learner may find this sequence helpful:

1. Read the ethics, authorization, and safety sections first.
2. Review the Kali Linux installation and first-boot guidance.
3. Build a local virtual lab.
4. Learn basic Linux commands and file navigation.
5. Study networking and traffic-analysis fundamentals.
6. Practice with beginner-friendly security tools.
7. Review vulnerability assessment and web testing concepts.
8. Study digital-forensics and incident-response sections.
9. Explore AI security, prompt testing, MCP, and RAG.
10. Read the AI Risk Management and Governance manual.
11. Study the ISO/IEC 42001 practical AIMS manual.
12. Practice only in authorized labs such as systems you own or approved training environments.

## Safe and Ethical Use

All technical material in this repository is intended for lawful, ethical, and authorized learning.

Use security tools and techniques only:

- On systems you own
- In isolated personal labs
- In approved training environments
- On systems for which you have explicit written authorization

Never use these materials to:

- Access systems without permission
- Collect private information without authorization
- Disrupt services
- Bypass access controls
- Test public targets without written approval
- Harm another person, organization, or network
- Hide unauthorized activity

**Technical ability does not create permission.**

Unauthorized access or testing can result in disciplinary action, loss of employment, civil liability, criminal penalties, and long-term damage to a cybersecurity career.

---

## Lab Safety Recommendations

Before practicing:

1. Use a virtual machine or isolated lab.
2. Keep snapshots and backups.
3. Do not bridge vulnerable machines directly to public networks.
4. Use intentionally vulnerable training targets.
5. Do not store real sensitive data in the lab.
6. Keep testing records and authorization documentation.
7. Stop immediately if a test affects an unintended system.
8. Report accidental exposure or impact responsibly.
