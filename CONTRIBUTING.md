# Contributing to AI Security Manuals

Thank you for helping improve this public educational collection.

## Ways to Contribute

You may report or propose:

- Factual corrections
- Broken or outdated links
- Accessibility barriers
- Translation or terminology improvements
- Formatting problems
- Clarifications that improve beginner understanding
- Updates to standards, frameworks, products, or official guidance

## Before Submitting

1. Identify the manual, language, version, and affected section.
2. Explain what is incorrect or unclear.
3. Provide the proposed correction.
4. Cite primary or authoritative sources whenever possible.
5. Confirm that the submission contains no confidential, proprietary, personal, regulated, or production data.

## Safety Requirements

Contributions must not:

- Provide instructions for unauthorized access, disruption, monitoring, persistence, credential theft, or data collection
- Include credentials, secrets, private keys, tokens, personal data, or production information
- Misrepresent legal, regulatory, audit, certification, or professional requirements
- Promise employment, certification, compliance, security, or other guaranteed outcomes
- Copy third-party material without permission

Security-testing content must be limited to lawful, authorized, and clearly scoped environments.

## Translation Guidance

- Use natural professional language rather than literal word-for-word translation.
- Preserve technical meaning, warnings, limitations, and ethical safeguards.
- Use locale-appropriate terminology.
- Identify whether a fluent human reviewer has reviewed the submission.
- Do not describe machine-assisted translation as human-reviewed.

## Accessibility Guidance

Contributions should preserve:

- Logical heading order
- Meaningful link text
- Clear and concise language
- Properly identified table headers
- Alternative text for meaningful images
- Sufficient contrast
- Logical reading order
- Correct document-language settings

See [ACCESSIBILITY.md](ACCESSIBILITY.md) for the project workflow.

## Submitting an Issue

Open a GitHub issue and include:

- Manual title
- Language
- File type
- Version
- Page or section
- Description of the problem
- Proposed correction
- Supporting source

Do not include sensitive information in a public issue.

## Submitting a Pull Request

Keep each pull request focused on one correction or closely related set of changes. Explain what changed, why it changed, which sources support it, and how the result was checked.

Submissions may be edited, declined, or deferred to preserve accuracy, safety, accessibility, consistency, and project scope.
